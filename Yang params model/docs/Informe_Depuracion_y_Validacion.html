Resumen ejecutivo.

Tras la migración literal de MATLAB a Python (fase previa), el modelo corría pero arrastraba bugs heredados, parámetros esparcidos en múltiples archivos y sin forma de verificar que produjese resultados válidos. Este trabajo transformó ese código operativo pero frágil en un modelo robusto, totalmente configurable y autovalidable: corrección de 7 bugs críticos, centralización de todos los parámetros tunables en un único config.py, documentación biológica por módulo, una suite de validación de 25 tests y una reorganización completa de carpetas con documentación explicativa.

3. Contexto y necesidad

El modelo Yang Params simula la dinámica poblacional anual de dos especies de mosquitos vectores del dengue (A. aegypti y A. albopictus) mediante un sistema de 6 ecuaciones diferenciales ordinarias (ODE) con tasas biológicas dependientes de la temperatura. El objetivo último es calibrar los coeficientes de competencia interespecífica (beta_ba, beta_ab) contra datos de campo del INSPI tomados en la localidad LITA (2018–2021).

El estado previo era:

  • El código MATLAB original contenía bugs documentados (variable tspan indefinida, parámetro p no propagado al ODE, equilibrio sin validación).
  • La migración a Python replicó esos bugs línea a línea.
  • El código Python corría pero sin validación de que los números producidos fueran correctos — sólo "no-NaN".
  • Parámetros hardcodeados dispersos en 4+ archivos. Cambiar cualquiera requería buscar dónde estaba escondido.
  • La estructura modular existía pero sin docstrings: 25 archivos sin explicación del rol biológico, unidades o rango válido.

4. Objetivos del trabajo

  1. Depurar: que el código corra sin fallos silenciosos y con mensajes de error útiles.
  2. Desacoplar: que todos los parámetros tunables vivan en un único punto editable.
  3. Documentar: que cualquier persona pueda abrir un módulo y entender qué hace.
  4. Validar: que los outputs sean verificables y las regresiones detectables automáticamente.
  5. Ordenar: que cualquiera entienda qué es cada carpeta y qué presentar.

7. Interpretación de las gráficas

7.1 rates_vs_temperature.png — tasas biológicas

Qué representa: cada uno de los 14 polinomios biológicos del modelo evaluado directamente en temperatura. Permite validar visualmente que cada tasa tiene la forma cualitativa reportada en la literatura.

Cómo leerla (panel por panel):

PanelMuestraForma esperada
Fecundidad basephi_A, phi_BUnimodal con pico en zona tropical (25–30 °C); cae en los extremos.
Oviposición efectivaphi_p_a, phi_p_bMuy similar a fecundidad base (sigma > mu_F en rango operacional).
Desarrollo huevo→larvapsi1_a, psi1_bCreciente con T; aegypti con valores más bajos.
Desarrollo larva→adultopsi2_a, psi2_bAlrededor de 0.1–0.15/día; saturación.
Mortalidad huevomuE_a, muE_bAegypti baja y estable; albopictus alta en T<25 (artefacto del clipping).
Mortalidad larvamuL_a, muL_bTasas bajas, 0.02–0.1/día.
Mortalidad adultomuF_a, muF_bAegypti ~0.03/día. Picos en extremos = artefacto.
Sobre los extremos aparentemente anómalos. En T=15 o T=35 algunos polinomios cruzan cero y, tras el clipping de seguridad, se estabilizan en valores extremos. Esto no afecta al modelo porque el modelo de temperatura solo genera valores en ~24–27 °C. Los extremos aparecen aquí sólo porque evaluamos los polinomios fuera del rango de operación real, para testear su comportamiento.

7.2 model_fractions.png — dinámica del modelo

Qué representa: cómo se distribuye la población de cada especie entre sus tres etapas (huevo, larva, adulto) a lo largo del año. La fracción en cada instante normaliza por el total de la especie: Ea / (Ea + La + Fa).

Leyenda:

  • Rojo sólido — Fracción de huevos aegypti (Ea)
  • Verde punteado — Fracción de larvas aegypti (La)
  • Azul punto-raya — Fracción de adultos aegypti (Fa)
  • Cian sólido — Fracción de huevos albopictus (Eb)
  • Magenta punteado — Fracción de larvas albopictus (Lb)
  • Amarillo punto-raya — Fracción de adultos albopictus (Fb)

Lectura del resultado obtenido:

  • Aegypti: las tres fracciones se mantienen estables todo el año — el modelo está en equilibrio. Huevos ~79%, larvas ~13%, adultos ~8%, con pequeñas oscilaciones estacionales.
  • Albopictus: fracciones constantes (con E_b dominando porque arranca con 10 huevos y cero larvas/adultos) hasta el día 355, donde la especie se "introduce" y empieza a reproducirse. A partir de allí las fracciones se acomodan rápidamente a una nueva distribución.
  • La discontinuidad en día 355 es deliberada: modela la introducción de una especie invasora en un ecosistema con aegypti ya establecido.
Validación visual: ambas gráficas son consistentes con la dinámica documentada en el paper fuente y con los rangos esperados de Yang 2011 / Mordecai 2019. El test de regresión garantiza que estas formas no cambiarán sin intención en el futuro.

8. Protocolo de validación sin MATLAB

Como no disponemos de MATLAB para generar ground truth, la validación se sostiene en tres pilares complementarios:

8.1 Tests estructurales (13 tests)

python -m pytest tests/test_baseline.py -v

8.2 Tests contra rangos de literatura (11 tests)

python -m pytest tests/test_literature_ranges.py -v
  • phi_A pico en rango tropical (24–32 °C), valor 2–20 huevos/ciclo
  • muF_a en T óptimo: 0.01–0.1 /día (vida 10–60 días)
  • psi2_a/psi2_b: 0.05–0.25 /día
  • phi_A unimodal
  • psi1_a finito en todo el dominio — verifica el fix del bug

8.3 Test de regresión contra snapshot (1 test)

python -m pytest tests/test_regression.py -v
# o
python -m dengue_model.diagnostics --compare-snapshot

Cualquier cambio numérico por encima de rtol=1e-8 falla el test.

Puente a MATLAB. El CLI --compare-baseline ya está listo; sólo falta correr ELF.m con save('baseline_matlab.mat', 't', 'y') y convertir a JSON.

9. Cómo usarlo y editar parámetros

9.1 Correr la simulación

# Configuración por defecto
python -m dengue_model.ELF

# Sin gráficas
python -m dengue_model.ELF --no-plot

# Override de betas
python -m dengue_model.ELF --beta-ba 0.3 --beta-ab 0.7

# Con config externa
python -m dengue_model.ELF --config mi_experimento.json

9.2 Generar plantilla de configuración

python -c "from dengue_model.config import ModelConfig; ModelConfig().save_json('template.json')"

9.3 Uso programático

from dengue_model import CONFIG, ModelConfig, run_elf, set_config

CONFIG.beta_ba = 0.3
CONFIG.carrying_capacity = 2e5
from dengue_model.my_parameters import rebuild_P; rebuild_P()
result = run_elf(plot=False)

# O cargar desde archivo
set_config(ModelConfig.from_json("mi_experimento.json"))

9.4 Herramientas de diagnóstico

python -m dengue_model.diagnostics --plot-rates
python -m dengue_model.diagnostics --snapshot
python -m dengue_model.diagnostics --save-figure salida.png
